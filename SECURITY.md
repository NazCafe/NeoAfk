Security Policy

Thank you for helping keep NeoAfk secure.

The security of our users, contributors, and the wider Minecraft community is important. If you discover a security issue, please report it responsibly so it can be investigated and fixed before public disclosure.

---

Supported Versions

The table below indicates which versions currently receive security updates.

Version| Supported
Pre-Alpha (main)| ✅ Yes
Older development builds| ❌ No

As NeoAfk matures, only actively maintained releases will receive security fixes.

---

Reporting a Vulnerability

If you believe you have found a security vulnerability, please do not open a public GitHub Issue.

Instead:

1. Contact the project maintainer privately through GitHub.
2. Provide a clear description of the issue.
3. Include reproduction steps if possible.
4. Explain the potential impact.
5. Include logs or screenshots if they help demonstrate the issue.

Responsible disclosure gives us time to investigate and prepare a fix before the vulnerability becomes widely known.

---

What Is Considered a Security Issue?

Examples include:

- Remote code execution
- Authentication bypass
- Arbitrary file access
- Denial-of-service vulnerabilities caused by malformed packets
- Sensitive information leakage
- Unsafe handling of untrusted network data
- Dependency vulnerabilities with significant impact

General bugs, crashes, or feature requests should be reported through GitHub Issues instead.

---

Response Process

When a security report is received, the project maintainer will aim to:

1. Acknowledge the report.
2. Investigate and reproduce the issue.
3. Determine the severity.
4. Develop and test a fix.
5. Release the fix.
6. Publicly document the issue if appropriate after it has been resolved.

Response times may vary depending on the complexity of the issue and the maintainer's availability.

---

Security Best Practices

NeoAfk aims to follow secure development practices, including:

- Validating network input.
- Handling malformed packets safely.
- Avoiding crashes from unexpected data.
- Keeping dependencies up to date.
- Writing clear, maintainable code.
- Reviewing AI-assisted code before integration.
- Testing protocol changes before release.

---

AI-Assisted Development

NeoAfk is developed using an AI-assisted workflow.

While AI tools can improve productivity, all generated code is reviewed, tested, and validated before being merged into the project.

Security decisions are based on testing and code review—not solely on AI-generated suggestions.

---

Scope

This policy applies to:

- Source code
- Documentation
- Configuration files
- Build scripts
- Deployment examples
- Future plugins or extensions maintained within the NeoAfk repository

---

Out of Scope

The following are generally not considered security vulnerabilities for this project:

- Requests for new features
- Minor coding style issues
- Performance optimizations
- Unsupported Minecraft versions
- Issues caused by third-party server modifications unless they expose a vulnerability in NeoAfk itself
