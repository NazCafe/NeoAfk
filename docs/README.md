📈 Documentation Philosophy

NeoAfk's documentation follows several guiding principles:

- Clear — Easy to read and understand.
- Accurate — Matches the current implementation.
- Educational — Explains both how and why features work.
- Maintainable — Organized into focused documents.
- Transparent — Planned features are clearly marked.

Documentation is updated alongside the project whenever possible.

---

📝 Contributing to Documentation

Documentation improvements are always welcome.

You can contribute by:

- Fixing grammar or spelling.
- Improving explanations.
- Adding examples.
- Reporting outdated information.
- Writing new documentation for upcoming features.

Please keep documentation concise, accurate, and beginner-friendly where possible.
